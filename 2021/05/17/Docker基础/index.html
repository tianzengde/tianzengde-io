三、docker镜像

1.查看本地镜像

REPOSITORY：表示镜像的仓库源

TAG：镜像的标签

IMAGE ID：镜像ID

CREATED：镜像创建时间

SIZE：镜像大小
同一仓库源可以有多个 TAG，代表这个仓库源的不同个版本，如 ubuntu 仓库源里，有 15.10、14.04 等多个不同的版本，我们使用 REPOSITORY:TAG 来定义不同的镜像。

2.拉取镜像

我们的镜像都可以存放到Docker Hub网站(gitee码云)上，这是一个开源的镜像库

如果从这个网站上 下载东西 就可以使用 pull 拉取

如果没有添加版本tag 则下载默认的最新版本 mysql:latest

3.删除镜像

docker rm

如果 镜像被容器在使用，默认是不能删除的，可以通过 -f 强制删除

四、容器(container)操作

1.运行容器

在本地 如果有镜像 就可以运行该镜像 生成容器，如果镜像不存在 会去网上pull后在运行

–name 在启动的时候给容器定义一个名字

-d 后台运行容器

–interactive , -i 交互

–tty , -t 终端

这两个一般都是一起用 -it 就是交互式启动容器

2.查看容器

注意：此时虽然容器已经在运行中，但是镜像占用的端口是 Docker 容器内部的端口，并不是服务器的端口，所以必须做 端口映射 将服务器的实际端口映射到 Docker 容器中的端口才能访问。

3.启动容器

4.停止容器

5.删除容器

删除容器前，必须先停止容器运行，根据 容器 id 进行删除

注意：rm 参数是删除容器，rmi 参数是删除镜像

镜像运行在容器中，docker 中可以运行多个互补干扰的容器，可以将同一个镜像在多个容器中进行运行

6.端口映射

–name：自定义容器名，不指定时，docker 会自动生成一个名称
-d：表示后台运行容器
image-name：指定运行的镜像名称以及 Tag
-p 表示进行服务器与 Docker 容器的端口映射，默认情况下容器中镜像占用的端口是 Docker 容器中的端口与外界是隔绝的，必须进行端口映射才能访问

7.从容器中提取出镜像

如果我们给现有的容器做了修改，而且这个修改以后还要常用，我们就可以提取出镜像，作为一个新的版本
